# c-programming
Collection of C programs from beginner to advanced. Intended as reference for basic principles to advanced topics.

## Getting Started
C is a wonderful language that can be intimidating at first glance. The syntax itself looks complex enough to kill a terminal! Yet, when you understand _why_ C is such a powerful language, the dots start connecting and pieces start clicking together! 

### Why C is so good
There are many reasons as to why C is so good. Such reasons include: allow low-level programming to interact directly with hardware _without_ overhead of automatic memory management, high-level enough to remain universally portable across nearly every computing architecture. 
    1. Performance: C programs execute incredibly fast and predictably, because C lacks high-level abstractions like garbage-collection or dynamic object lookups. This makes the compiled code map directly to machine instructions allowing to easily outperform many modern languages. Which is also why we can compare something like assembly (machine) code to C code.
    2. Hardware Control: Developers have the ability to explicitly allocate, manipulate, and free memory down to the exact byte. This kind of control and precision is necessary for creating highly optimized software, including software in the fields of IoT and Robotics firmware, game engine components, and embedded systems.
    3. Foundation of Modern OS: Core kernels of all major OSs (Linux, Windows, macOS, Android, and iOS) are all written in C, forming the backbone of the world's digital infrastructure.
    4. Simplicity & Small Grammar: C specification is small and intuitive. The straightforward syntax (when understood) compiles quickly and does not require a steep learning curve to understand CS basics, which makes it an excellent teaching tool.

### How to properly learn the C language
The advancement of the internet has brought about an abundance of resources readily available to EVERYONE and the amount of resources can become easily overwhelming, especially for beginners. The best way to learn virtually _any_ programming language comes from practicing. By practicing, I do not mean watching and copying the code that the developer is doing. But rather, start with the basics. Learn how to output to the console. How to take user input. What each data type is and how to properly use it. Why is memory so important in a language like C? What are loops and how to use them? Why are strings in C just arrays of Char data types? 

When you start to ask yourself these questions and _build_ small programs to reinforce your learning, that is where the _real_ learning begins. Write code. Compile it. Run it. Doesn't work? Start debugging and troubleshooting. Change the code. Break it on purpose. See what changing certain lines of code does. All of this is what will get your brain connecting the dots and gears will start moving!
